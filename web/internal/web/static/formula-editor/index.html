The Formula Editor

The approved design for issue #445: a gamified, fully working editor for .agentfactory/store/formulas/*.formula.toml. D (Foundry) structure, B's structure-derived rarity, the console's Neon Bazaar theme token-for-token. The TOML text is the single source of truth: every visual edit is a comment-preserving minimal text patch, validation mirrors the engine's internal/formula rules, and in Chromium it opens and saves the real store files.

The identity

World
The factory floor at night — the console's Neon Bazaar. A formula is a living assembly line of stations; gates are inspection arches; saving is pouring the cast.
Signature
Test Fire — a spark runs the computed DAG while the four inspection lamps land — plus rarity frames computed from structure: gate → legendary (amber), merge → epic (violet), wired → rare (cyan), root → common.
Palette
Token-for-token the console's variables.css. Chakra Petch / Sora / Space Mono.
Risk
A real parser + comment-preserving patch engine inside a design prototype — taken because the issue's core complaint was trust, and trust needed a working round-trip to judge. The review confirmed it: "this is the binding architecture going forward."
Motion
Event-only: Test Fire, lamp changes, slips. prefers-reduced-motion collapses all of it.

How to drive it — the round-trip demo

  1. Open the editor. It boots on a byte-exact demo copy of ultra-review.formula.toml — walk the 16-station line, click a station, raise its panel.
  2. Real-file round trip (Chromium): Open a real .toml… → .agentfactory/store/formulas/ultra-review.formula.toml. Retitle a station, open Maintenance hatch → Diff vs file — one changed line. Pour & save, then git diff: exactly that line. Comments and formatting untouched.
  3. Load the convoy demo (six legs merging into synthesis) and the web-design demo (22 stations, five legendary arches) from the demo picker.
  4. Break something: unplug a belt or mangle the TOML in the hatch — lamps, fault list, locked pour. Press F to Test Fire.
  5. Headless proof of the engine against all 28 store formulas:
node .designs/web-ui/final/scripts/test-engine.js

No File System Access API (Firefox/Safari)? Everything works on demo copies, with Paste TOML… and Download as the fallback loop. No af write verbs anywhere — authoring stays in files, af remains the operating plane.
